Every TOML file under `/etc/hal0/` is validated against Pydantic v2 models at
load time. A typo like `device = "gpu-rcom"` raises a `ValidationError` with
the offending field path rather than failing later at dispatch.

## File locations (FHS)

When `HAL0_HOME` is unset, hal0 uses the standard layout. Setting
`HAL0_HOME=/some/path` re-roots everything under that prefix (used for dev
installs and tests).

| File | FHS path | Purpose |
|---|---|---|
| `hal0.toml` | `/etc/hal0/hal0.toml` | Top-level config. |
| `slots/<name>.toml` | `/etc/hal0/slots/<name>.toml` | One file per slot. |
| `profiles.toml` | `/etc/hal0/profiles.toml` | Profile catalog (optional; seeds used if absent). |
| `providers.toml` | `/etc/hal0/providers.toml` | External LLM providers. |
| `upstreams.toml` | `/etc/hal0/upstreams.toml` | Slot + remote upstream catalog. |
| `capabilities.toml` | `/etc/hal0/capabilities.toml` | Capability-slot selections. |
| `agents/<name>.toml` | `/etc/hal0/agents/<name>.toml` | One file per bundled/user agent. |
| `hardware.json` | `/etc/hal0/hardware.json` | Cached hardware probe (written by `hal0 probe`). |
| `manifest.json` | `/etc/hal0/manifest.json` | Release manifest (pinned toolbox image digests). |

State (registry, models, slot state, memory banks) lives under
`/var/lib/hal0/`. Config and state are both preserved across updates.

## Valid enums

| Set | Values |
|---|---|
| `device` (hardware preference) | `gpu-rocm`, `gpu-vulkan`, `cpu`, `npu` |
| `provider` (engine label) | `llama-server`, `flm`, `kokoro`, `comfyui` |
| slot `type` (dispatch routing) | `llm`, `embedding`, `reranking`, `transcription`, `tts`, `image` |
| seed profiles | `rocm`, `rocm-dnse`, `rocm-moe`, `vulkan`, `flm`, `tts`, `comfyui` |

<Aside type="caution">
Use `device`, not the legacy `backend` field. `backend` is deprecated: a slot
TOML that sets `backend` without `device` logs a deprecation warning and the
loader auto-fills `device`. Set `device` directly. NVIDIA/CUDA is unsupported —
it is not a valid device.
</Aside>

## `hal0.toml`

The top-level config is assembled from these sections, each with sane
defaults (an empty file is valid).

### `[meta]`

| Key | Type | Default | Notes |
|---|---|---|---|
| `schema_version` | int (≥1) | `1` | Bumped by `hal0 config migrate`. |

### `[slots]` — global slot policy

| Key | Type | Default | Notes |
|---|---|---|---|
| `max_slots` | int (≥0) | `0` | Max concurrent slots; `0` = unlimited. |
| `port_range_start` | int | `8081` | First port in the slot pool. |
| `port_range_end` | int | `8200` | Last port (inclusive); must be ≥ start. |
| `idle_timeout_s` | int (≥0) | `300` | Default idle-eviction TTL; `0` disables. Per-slot value overrides. |

### `[dispatcher]`

| Key | Type | Default | Notes |
|---|---|---|---|
| `prefetch_timeout_s` | float (>0) | `8.0` | Cold-cache prefetch timeout. |
| `prefetch_parallel_cap` | int (≥1) | `4` | Max concurrent upstream prefetches. |

### `[telemetry]`

| Key | Type | Default | Notes |
|---|---|---|---|
| `enabled` | bool | `false` | Opt-in anonymous telemetry. |
| `channel` | `"stable"` \| `"nightly"` | `"stable"` | Update channel. |

### `[models]` — discovery

| Key | Type | Default | Notes |
|---|---|---|---|
| `roots` | list[str] | `["/var/lib/hal0/models"]` | Absolute roots scanned for model files. |
| `auto_scan_on_start` | bool | `true` | Run discovery scan at startup. |
| `file_extensions` | list[str] | `[".gguf", ".safetensors"]` | Candidate file suffixes. |
| `store` | str | `""` | Single source of truth for where models are read/written. When set, the pull engine writes here and slot containers bind-mount it. `HAL0_MODEL_STORE` overrides. |

<Aside type="note">
`store` is the canonical model-store field. When empty, hal0 falls back to the
deprecated `pull_root` (default `/var/lib/hal0/models`). Point `store` at your
fast disk / network mount to align pulls and slot bind-mounts on one path.
</Aside>

### `[memory.graph]` — graph extraction (ADR-0014)

| Key | Type | Default | Notes |
|---|---|---|---|
| `enabled` | bool | `false` | Off by default. When false, `memory_add` skips graph extraction; vector search still works. |
| `route` | `"upstream"` \| `"primary"` \| `"agent"` | `"upstream"` | Where the extraction LLM call is dispatched. |
| `upstream.provider` | str | `"openrouter"` | Provider id; must match a `providers.toml` entry for key resolution. |
| `upstream.model` | str | `""` | Model id. Required when `enabled = true` and `route = "upstream"`. |

## Per-slot TOML (`slots/<name>.toml`)

A slot config has `[slot]`, `[model]`, and optional `[server]` / `[npu]` /
`[image]` sections. Unknown keys round-trip (the model uses `extra="allow"`).

### `[slot]` section

| Key | Type | Default | Notes |
|---|---|---|---|
| `name` | str | required | Lowercase alphanumeric + `-`/`_`, starts alphanumeric, ≤32 chars. |
| `port` | int | required | Host port in range `8081–8200` (127.0.0.1 only). |
| `type` | str | `llm` | Dispatch routing: `llm` \| `embedding` \| `reranking` \| `transcription` \| `tts` \| `image`. |
| `device` | str | `gpu-rocm` | `gpu-rocm` \| `gpu-vulkan` \| `cpu` \| `npu`. |
| `provider` | str | `llama-server` | Engine label (round-trips for compatibility/UI). |
| `profile` | str \| null | `null` | Profile name from `profiles.toml` (image + tuned flags). |
| `role` | str \| null | `null` | Role hint: `primary`, `utility`, `npu`. Derived from name when unset. |
| `enabled` | bool | `true` | Start this slot on hal0 startup. |
| `enable_thinking` | bool \| null | `null` | Per-slot reasoning default; overridable per request. |
| `mtp` | bool \| null | `null` | Per-slot multi-token-prediction override (effective only on `rocmfp4`/MTP-capable profiles). |
| `chat_template` | str \| null | `null` | Chat-template id override (`auto`/null = GGUF-embedded). |
| `workers` | int (≥1) | `1` | Parallel request workers. |
| `idle_timeout_s` | int (≥0) | `300` | Idle TTL for this slot; `0` disables. |

<Aside type="note">
`backend` (deprecated) and `runtime` (always `container`) still round-trip but
should not be authored. Slots always run as podman containers managed by the
container provider.
</Aside>

### `[model]` section

| Key | Type | Default | Notes |
|---|---|---|---|
| `default` | str | `""` | Default model id from the registry. |
| `context_size` | int (≥128) | `null (derived)` | When unset, hal0 derives the context window from the model's native size (capped) or a safe floor; never silently falls back to 4096. |
| `n_gpu_layers` | int | `-1` | Layers offloaded to GPU; `-1` = all. |
| `rope_freq_base` | float (≥0) | `0.0` | RoPE base override; `0.0` = model default. |

### `[server]` section

| Key | Type | Default | Notes |
|---|---|---|---|
| `extra_args` | str \| null | `null` | Freeform `llama-server` CLI passthrough; tokenised and merged with the profile/model flags. |

### `[npu]` section (NPU slots)

| Key | Type | Default | Notes |
|---|---|---|---|
| `asr` | bool | `false` | Enable ASR via FLM `--asr 1`. |
| `embed` | bool | `false` | Enable embeddings via FLM `--embed 1`. |

### `[image]` section (img / ComfyUI slot)

| Key | Type | Default | Notes |
|---|---|---|---|
| `idle_restore_minutes` | int (≥0) | `60` | Idle minutes before the GPU arbiter restores stopped LLM GPU slots; `0` = never. |
| `default_size` | str | `"1024x1024"` | Default output size (WxH). |
| `default_steps` | int (≥0) | `0` | Default sampler steps; `0` = model-class default. |

### Example slot TOML

<Tabs>
<TabItem label="GPU chat slot">

```toml
[slot]
name = "chat"
port = 8081
device = "gpu-rocm"
profile = "rocm"
role = "primary"

[model]
default = "qwen3-4b-q4_k_m"
context_size = 32768

[server]
extra_args = "-fa on"
```

</TabItem>

```toml
port = 8090
device = "npu"

[model]
default = ""
```

## Profiles (`profiles.toml`)

A profile is a reusable backend template: a container image plus a bench-tuned
flag bundle. Slots reference a profile by name; the profile supplies the image
and flags, the slot supplies the model, context size, and port. When
`profiles.toml` is absent, hal0 serves the built-in seed profiles.

| Key | Type | Notes |
|---|---|---|
| `image` | str | Container image ref (required). |
| `flags` | str | Bench-tuned `llama-server` flags (no model/port/ctx args). |
| `mtp` | bool | Append the MTP draft-speculation bundle at resolve time. |
| `device_class` | `gpu` \| `cpu` \| `npu` \| `img` | Device class the profile targets. |
| `backend` | `rocm` \| `vulkan` \| null | GPU runtime (authoritative ROCm-vs-Vulkan choice); null for non-GPU profiles. |
| `intent` | str | Human label shown in the dashboard. |
| `quant` | str | Weight quant format shown as a chip (informational). |

Seed profiles: `rocm`, `rocm-moe`, `rocm-dnse`, `vulkan` (GPU), `flm` (NPU),
`tts` (CPU / Kokoro), `comfyui` (image generation). `rocm` is the `gpu-rocm`
default (MoE/agents, FP4, no MTP).
