The five [built-in slots](/docs/slots/built-in-slots/) cover the
OpenAI-compatible surface. You can define additional slots for a
second chat model, a vision model, the FLM provider on AMD XDNA,
ComfyUI, or anything else a provider supports.

## When you want a custom slot

- **Keep two chat models hot at once.** A small fast model for
  autocomplete, a big slow model for reasoning. Different slot names,
  same `/v1/chat/completions` endpoint, addressed by model name.
- **Use the NPU alongside the iGPU.** A custom `npu` slot binds the
  FLM provider to XDNA so the iGPU stays free for the `primary` slot.
- **Dedicated vision slot.** A multimodal model lives in its own slot
  so swapping the chat primary doesn't unload it.

<Aside type="caution">
  Slot ports are allocated from `8081`–`8099`. The dashboard form will
  refuse to create a slot whose model exceeds the probe's reported
  VRAM/GTT headroom unless you confirm the override.
</Aside>

## Defining one (dashboard)

The dashboard's Slots view has an **Add slot** form. It takes:

- A name (must match `^[a-z][a-z0-9_-]*$`).
- A provider (`llama.cpp`, `flm`, `moonshine`, `kokoro`, `comfyui`).
- A model registry ref.
- An idle-timeout, after which the slot transitions `ready → idle`
  and becomes an unload candidate.

The form writes the slot definition to the TOML config atomically
(see [Config](/docs/operate/config/)), then stages the systemd unit.
You start the slot with the **Load** button.

## Defining one (CLI / config)

Slots are entries in `hal0.toml` under `[slots.<name>]`. The shape
mirrors the dashboard form one-for-one. After editing:

```sh
hal0 config validate    # schema check
hal0 config show        # confirm the merged view
hal0 slot load my-slot  # bring it up
```

See [Config schema](/docs/reference/config-schema/) for the field
list.

## Limits

- Slot names must be unique. The five built-in names
  (`primary`, `embed`, `stt`, `tts`, `img`) are reserved, as is the
  auto-managed `embed-rerank` slot synthesised by the embed
  capability.
- Total slot count is capped by the port range (`8081`–`8099`, so
  19 concurrent slots).
- A slot can only host one model at a time. To run two models on the
  same provider, define two slots.
