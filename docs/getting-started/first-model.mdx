The one-liner installer seeds the recommended Main slot and writes the
first-run sentinel automatically — no synchronous download, no
interaction required. When you're ready to customize, run:

```sh frame="terminal"
hal0 setup
```

This interactive TUI walks you through every choice. You can re-run it
at any time to reconfigure storage, swap models, or toggle Extensions.

## What `hal0 setup` does

<Steps>
1. **Storage.** Confirm or change the directory where models are stored
   (default: `/var/lib/hal0/models`). If you have an existing model
   store on a NAS or separate volume, point `--storage-dir` here.

2. **Extensions.** Choose which optional components to install:
   - **Apps** — Open WebUI (chat UI at `:3001`, zero config, already
     pointed at your Main slot).
   - **Agents** — Hermes-Agent (service shape) or pi-coder (CLI shape).
     `hal0 setup` auto-wires whichever you pick into the platform.

3. **Main model.** Pick the chat/LLM model for the primary slot.
   `hal0 setup` shows curated options filtered to what fits your
   detected hardware (GPU VRAM / unified RAM).

4. **Agent model.** Optionally pick a dedicated model for the agent
   slot (smaller / faster). Defaults to the Main model if skipped.

5. **NPU opt-in** *(Strix Halo only).* Enable the FLM trio — a single
   NPU container hosting chat + transcription + embedding concurrently
   on the AMDXDNA hardware context. Requires FastFlowLM to be present.

6. **Download.** Models download with live progress — bytes, percent,
   speed, and ETA per model. Downloads run in the background; you can
   exit and the pulls continue.
</Steps>

## Non-interactive mode

```sh frame="terminal"
# Fully automatic — recommended defaults, no prompts
hal0 setup --auto

# Seed slots without downloading models now
hal0 setup --auto --no-pull

# Skip Extensions (Apps/Agents)
hal0 setup --auto --no-extensions

# Custom storage directory
hal0 setup --storage-dir /mnt/ai-models
```

The one-liner `curl hal0.dev/install.sh | bash` internally runs
`hal0 setup --auto --no-pull --no-extensions` to seed the Main slot
non-interactively. Set `HAL0_SKIP_SETUP=1` before invoking the
installer to skip first-run setup entirely.

## After setup

Once `hal0 setup` completes, check slot state with:

```sh frame="terminal"
hal0 status
```

`primary` (and any other enabled slots) should be `ready`. If a model
is still downloading, the slot stays `starting` until the pull
finishes.

<Aside type="note" title="Skipping setup — configure manually">
You don't have to use `hal0 setup`. Drive everything from the CLI:

```sh
hal0 slot load chat --model phi-3-mini-4k-instruct-q4
```

The slot manager handles the pull, the systemd unit, and the health
probe. `hal0 status` shows the same state machine `hal0 setup` streams.
</Aside>

## When a pull fails

Common causes and fixes:

- **No disk space.** Free the space and re-run `hal0 setup` — partial
  downloads resume.
- **Hugging Face rate-limit.** Set `HF_TOKEN` in `/etc/hal0/api.env`
  (or dashboard Settings → Secrets) and retry.
- **License not accepted.** Some gated models require you to accept
  terms on the Hugging Face model page before the API will serve files.
  Accept there, then retry.

## Changing models later

After setup the **Models** page in the dashboard is where you live:

```sh frame="terminal"
hal0 model list
hal0 slot swap chat --model qwen2.5-coder-7b-instruct-q4_k_m
```

See [recommended loadouts](/docs/hardware/strix-halo/#recommended-loadouts)
for a hardware-by-hardware breakdown of what fits where.
